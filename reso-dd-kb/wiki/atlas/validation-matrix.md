# Atlas Target - Validation Matrix (PR0.7)

Generated by `reso-dd-kb/scripts/build_validation_matrix.py`.

Validates every field in `wiki/atlas/atlas-target.dbml` (PR0.5) against:
- canonical RESO Data Dictionary 2.0 (`raw/fields.csv`)
- adoption metrics (`raw/field_metadata.csv`)
- live Atlas data samples (via Supabase MCP)

**Verdict legend**:
- **PASS** - canonical match; PR may proceed.
- **PASS-WITH-NOTE** - semantic match with a documented quirk; PR may proceed but the note must appear in the migration commit message + reso_field_descriptions caption.
- **FAIL** - Definition contradicts intent OR data is dirty (e.g., orphans). BLOCKS the dependent PR until the design is revisited and/or the data cleaned.

Per Principle 8 of the plan: no schema-touching commit lands without all three artefacts (canonical DBML, target DBML, this matrix) green for every field it touches.
