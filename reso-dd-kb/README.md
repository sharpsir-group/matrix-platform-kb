# reso-dd-kb

A small, self-contained RESO Data Dictionary 2.0 knowledge base for
LLM agents working on the Matrix MLS / Atlas platform. Built using
Karpathy's "LLM Wiki" pattern: immutable Raw inputs, an LLM-owned
generated Wiki, and a hand-written Schema (`AGENTS.md`) that turns any
LLM into a disciplined RESO librarian.

## What's in here

| Folder | Purpose |
|---|---|
| `raw/` | Immutable upstream sources — RESO DD 2.0 spreadsheet, parsed CSVs, RESO certification stats, our description corpus, scraped DDwiki metadata. |
| `wiki/resources/` | One Markdown page per RESO Resource. Generated by `scripts/refresh.py`. |
| `scripts/` | `refresh.py`, `fetch_field_metadata.py`, `dump_descriptions.sh`. |
| `AGENTS.md` | Schema the LLM obeys (boundaries, query workflow, refresh workflow, citation style). |
| `methodology.md` | One-page primer: naming, keys, lookups, lifecycle, how to read Sys%/Org%. |

## Why "descriptions only + DDwiki link-out"?

We rejected mirroring the full DDwiki prose. Size comparison:

| Strategy | Repo size | Refresh story | Verdict |
|---|---|---|---|
| **Descriptions only (this KB)** | ~5 MB | seconds (DB dump + xlsx parse) | **Adopted** |
| Lean MD mirror (per-field cleaned ddwiki) | ~12 MB | ~10 min/refresh + monthly cron | Revisit when offline-LLM matters |
| Full MD mirror | ~30 MB | ~10 min/refresh | Diminishing returns |
| Raw HTML mirror | ~240 MB | needs Git LFS | Not recommended |

Each Resource page embeds the short canonical description plus the
structured KV (BEDES, PropertyTypes, dates). When the agent needs deep
prose, it clicks the DDwiki link in the Fields table.

## Refresh

```bash
# 1. drop new RESO DD xlsx into raw/
# 2. parse xlsx -> fields.csv + lookups.csv (one-off openpyxl script)
# 3. scrape DDwiki for structured metadata + Sys%/Org%
python3 reso-dd-kb/scripts/fetch_field_metadata.py

# 4. dump description corpus from Atlas Supabase
PSQL_URL=postgres://... bash reso-dd-kb/scripts/dump_descriptions.sh

# 5. rebuild wiki/
python3 reso-dd-kb/scripts/refresh.py
```

`fetch_field_metadata.py` is incremental — `If-Modified-Since` keeps
steady-state refreshes touching ~30-50 pages instead of ~1,750.

## Using it (humans + LLMs)

- **Humans**: open `wiki/resources/<Resource>.md` and search the Fields
  table. Click the DDwiki link in the right-most column for full
  RESO prose.
- **LLMs**: read `AGENTS.md` first. It defines page anatomy, query
  workflow, citation style, and the strict "never invent a field"
  rule.
